# 有物之间小程序本地 Mock 自测清单

当前版本只测试微信小程序本地 mock 数据，不接 Supabase。

## 准备

- 用微信开发者工具打开项目根目录。
- 首次测试建议清理小程序本地缓存后重新编译，确认最新 mock 商品已写入本地 storage。
- 如需测试 owner 操作，先进入“我的”页模拟登录。默认 mock 用户是 `CUHK Student`。

## 二手商品

- 浏览商品：进入“二手”tab，默认只看到“在售”商品，两列瀑布流上下错落。
- 状态筛选：切换“全部 / 在售 / 已售 / 已下架”，确认列表状态符合筛选。
- 搜索商品：搜索标题、描述、分类，例如 `显示器`、`教材`、`厨房用品`。
- 搜索空结果：输入不存在的关键词，确认出现空状态提示。
- 清空搜索：点搜索框右侧清空按钮，确认恢复默认在售列表。
- 无图片商品：查看无图片商品卡片和详情页，确认占位图正常。
- 超长标题：查看超长标题商品，确认首页最多两行，详情页可完整查看。

## 发布与编辑

- 发布商品：进入发布页，填写标题、价格、分类、成色、地点、描述后提交。
- 价格为空：价格留空提交，确认提示“请填写价格”。
- 价格非数字：输入非法价格，确认提示“价格只能填写数字”。
- 图片上传：上传 1-9 张图片，确认不会超过 9 张。
- 编辑商品：登录后进入自己发布的商品详情，点“编辑商品”，修改后保存。
- 非本人商品不能编辑：登录后打开非 `CUHK Student` 发布的商品，确认不显示管理按钮；手动进入编辑页也会被拦截。

## 商品状态

- 下架商品：登录后打开自己的商品，点“下架”，确认状态变为“已下架”。
- 标记已售：登录后打开自己的商品，点“标记已售”，确认状态变为“已售出”。
- 删除商品：登录后打开自己的商品，点“删除”，确认出现二次确认，确认后首页不再显示该商品。
- 已售商品：切换首页“已售”，确认能看到已售商品。
- 已下架商品：切换首页“已下架”，确认能看到已下架商品。

## 联系与举报

- 查看联系方式：未登录时详情页显示“登录后查看联系方式”。
- 模拟登录：点详情页登录按钮，登录后返回详情页，确认显示微信号。
- 复制微信号：点“复制微信号”，确认 toast 提示复制成功。
- 我联系过的商品：复制微信号后进入“我的”，确认商品出现在“我联系过的商品”。
- 举报：点“举报”，选择原因，确认出现“举报已提交”提示。

## 收藏与我的

- 收藏商品：详情页点“收藏”，确认按钮变为“已收藏”。
- 取消收藏：再次点击，确认取消收藏。
- 我的收藏：进入“我的”，确认收藏商品出现在“我的收藏”。
- 我的发布：登录后进入“我的”，确认自己发布的商品全部展示，包括已售、已下架。
- 退出登录：点“退出登录”，确认用户信息、我的发布和联系记录按未登录状态展示。

## 通过标准

- 页面跳转不卡死。
- 商品列表、详情、发布、编辑、状态变更、收藏、联系记录都使用本地 mock 数据正常闭环。
- 非 owner 看不到管理按钮，也无法直接编辑非本人商品。
- 清理缓存后重新进入，小程序仍能加载默认 mock 商品。
